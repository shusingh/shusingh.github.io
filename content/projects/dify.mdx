## Change intent

This PR cleans up places where Dify used `Optional[str] = None` even though the consuming code only checked truthiness. In those paths, `None` was not carrying meaningful state. It was just widening the type surface.

The change moves those defaults to empty strings in OAuth and Swagger-related code paths, then updates tests to match the actual contract.

## Requirements

- Preserve existing runtime behavior.
- Avoid changing OAuth semantics.
- Keep Swagger generation output stable except for the cleaner default contract.
- Update unit and integration tests around the touched paths.
- Make future refactors less likely to treat `None` as a meaningful state.

The implementation audits each `Optional[str] = None` usage before changing it. The important question is whether `None` means something distinct from an empty string. If not, the type should not imply a state the code does not actually handle.

## Design decisions

**Preserve truthiness behavior.** The goal is contract cleanup, not logic change.

**Keep auth paths conservative.** OAuth code is not a place for clever refactors. The PR stays narrow and test-backed.

**Update tests with the code.** Type cleanup is easy to under-test because the runtime behavior appears unchanged. The tests document the intended contract.
