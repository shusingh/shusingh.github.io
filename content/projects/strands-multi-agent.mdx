## System intent

This repo is a focused workbench for multi-agent patterns: planning, execution, critique, and refinement. The goal is not to build a large product surface. The goal is to isolate agent coordination patterns and make the state transitions inspectable.

The core opinion is that agent systems should behave like software systems. Roles should be narrow, state should be explicit, and feedback loops should be bounded.

## Requirements

- Each agent role should have a narrow contract.
- Intermediate outputs should be inspectable before the next stage runs.
- Critic findings should be structured, not prose buried in a response.
- Human feedback should update canonical state instead of appending to chat history.
- The loop should be easy to test with repeatable cases.

The planner creates a structured plan. The executor performs the next bounded action. The critic evaluates the result against a rubric. The refiner applies feedback to the current state and hands it back into the loop.

## Design decisions

**State is the memory layer.** Chat history is useful for UX, but it is a weak source of truth. This repo pushes durable task state into explicit objects that can be inspected and diffed.

**Critic output is structured.** A critic that returns a paragraph is hard to act on. A critic that returns findings with section, severity, reason, and suggested fix can drive a real workflow.

**Small agents beat broad agents.** A planner does not need to execute. An executor does not need to judge. Keeping roles narrow makes each prompt easier to test.

## What I would improve

The next step is an eval harness that runs the same tasks across prompt and model changes. Agent loops feel impressive when they work once; they become engineering only when regressions are measurable.
