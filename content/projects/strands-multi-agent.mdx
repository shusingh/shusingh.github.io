## Why I built it

The fastest way to get better at agent architecture is to isolate the patterns. This repo is a place to test small multi-agent loops without the noise of a full product surface.

The focus is on state handoff, role separation, and evaluation. I am less interested in whether an agent can produce a neat answer once. I care about whether the loop is inspectable, repeatable, and resilient when an intermediate step is wrong.

## What it explores

- Planner agents that produce structured work plans instead of prose.
- Executor agents that operate on a constrained task surface.
- Critic agents that return structured findings.
- Shared state that can be inspected between each step.
- Feedback loops that can be bounded and tested.

## Next improvements

The next useful layer is an evaluation harness. Agent loops are easy to admire subjectively, but hard to improve without repeatable test cases and failure labels.
