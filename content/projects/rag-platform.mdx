## Why I built it

Most RAG examples stop at "embed a document and ask a question." The interesting work starts after that: indexing strategy, metadata filters, citation assembly, cache invalidation, query traces, and the path from a user question to a source-backed answer.

This repo is my workbench for those production concerns. The goal is not to build a polished SaaS product. The goal is to keep sharpening the systems instincts behind retrieval-heavy AI products.

## What it explores

- Go service boundaries around ingestion, retrieval, and answer assembly.
- OpenSearch for hybrid retrieval and metadata-aware search.
- Postgres for durable records and query trace state.
- Redis for cache paths where repeated questions should not pay the full retrieval cost every time.
- Citation-first response shape, where the system treats evidence as part of the output instead of a footnote.

## Next improvements

The pieces I would keep pushing are query evaluation, reranking, and observability. A retrieval platform is only as good as the evidence it retrieves, and the only honest way to improve that is to make failures visible.
