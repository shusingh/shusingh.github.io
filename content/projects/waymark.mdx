## System intent

Waymark is a local-first AI memory CLI for personal agents. It captures messy
thoughts, imports explicit files, tracks decisions, saves reflections, and
answers questions over the user's own context with visible sources.

The product constraint is privacy. Personal memory is not a casual data stream.
Waymark stores data locally in SQLite, makes AI optional, and avoids background
folder scans or hidden indexing jobs. It is designed as an agentic memory layer:
useful to a human in the terminal today, and structured enough to become
grounded context for local AI agents later. The CLI can be installed from PyPI
as `waymark-memory`, while the command users run stays simple:

```bash
waymark
```

## Requirements

- Work offline for manual capture, timeline review, decisions, and backups.
- Store user data in a local SQLite database, not a hosted service.
- Make file imports explicit and preview-first.
- Answer questions only from saved memories and imported sources.
- Keep AI optional through local Ollama integration.
- Shape memory into retrieval-ready context for future AI agent workflows.
- Package the tool like real software: docs, tests, releases, GitHub artifacts,
  and PyPI publishing.

## High-level design

The project is a Python CLI/TUI built around a local data model. Typer owns the
command surface, Textual provides the guided terminal app, and SQLite is the
source of truth for entries, tags, sources, reflections, decisions, and backup
metadata.

The important boundary is between memory and generation. Waymark can draft a
structured card or use local embeddings, but the durable record is still the
user-owned memory entry. Generated summaries and agent answers are not allowed
to become ungrounded narrative. They need saved sources.

## Design decisions

**Local-first by default.** The default storage model is a local Waymark home
directory. The tool does not require a hosted account to be useful.

**Explicit import over ambient indexing.** Files and folders are never scanned
automatically. Folder import is preview-first, bounded, resumable, and visible
about duplicates or skipped files.

**SQLite as the product spine.** The database is not an implementation detail.
It is what makes the memory trail inspectable, portable, and testable.

**Citations before fluency.** `waymark ask` is designed to retrieve relevant
saved memories and show sources. If nothing matches, the correct behavior is to
say so rather than fill the gap.

**Agent-ready memory, not agent hype.** The tool does not pretend to be an
autonomous assistant. It builds the substrate an AI agent needs first: durable
state, explicit sources, retrieval, citations, and user-owned context.

**Release discipline matters.** The project ships with strict typing, test
coverage, package smoke tests, docs builds, GitHub Releases, and Trusted
Publishing to PyPI. That release machinery is part of the product quality, not
an afterthought.

## Publishing path

The public release exposed a surprisingly useful packaging lesson: the PyPI
package name and the command name do not have to match. The `waymark` name was
already taken on PyPI, so the distribution publishes as `waymark-memory` while
the installed command remains `waymark`.

The final publish used PyPI Trusted Publishing instead of a long-lived token.
That keeps credentials out of the repository and ties the upload to a specific
GitHub workflow, repository, tag, and environment.

## What I would improve

The next phase is a better retrieval layer for personal AI agents: tighter
semantic search, clearer source ranking, and stronger review flows for memories
that came from imported documents. I would also like the TUI to become the main
daily surface rather than a wrapper around commands, with richer keyboard-first
workflows for capture, decision review, and weekly reflection.
