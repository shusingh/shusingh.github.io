## Why I built it

I use Java and Python heavily at work, and Go often on side projects. Rust is useful for stretching a different part of the backend brain: ownership, explicit error handling, and compile-time pressure toward cleaner boundaries.

This is intentionally a small API. The value is in practicing clean endpoint design and typed data flow without hiding behind framework magic.

## What it covers

- REST endpoints for blog-post style resources.
- Actix Web routing and request handling.
- Rust data models for request and response shapes.
- A compact service layout that is easy to extend.

## What I would add next

The next useful additions would be persistence, structured logging, and integration tests around the API contract.
