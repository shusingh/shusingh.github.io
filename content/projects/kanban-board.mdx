## What it is

A small but deliberate Kanban board: columns, draggable cards, a persistent state model, and a component architecture that stays clean as the feature set grows.

The point was not to build a Trello competitor. The point was to practice the parts of frontend that get lazy when you only ever build forms: drag-and-drop ergonomics, optimistic state updates, and component boundaries that survive when a feature requires three components to talk to each other.

## What it covers

- Drag-and-drop column reordering and card movement
- Local persistence so the board state survives a refresh
- Typed state model with explicit transitions, no `any`
- Responsive layout that works on tablet sizes without separate code paths
